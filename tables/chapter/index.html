    Chapter Table Migration

    The Chapter table represents the transformation from MySQL's Donor_Group table (where main = 1) to PostgreSQL's chapter table, serving as the foundation for managing fraternity chapters, alumni associations, and other chapteral entities.

    Overview

    Chapters are the primary entities that manage chapters, events, and member relationships. The migration enhances the original MySQL structure with improved data types, better international support, and enhanced audit capabilities.

    Schema Transformation

    MySQL Source Table: Donor_Group (main_group=1)

    CREATE TABLE `Donor_Group` (
  `id` int(10) unsigned NOT NULL AUTO_INCREMENT,
  `title` varchar(45) NOT NULL DEFAULT '',
  `main_group` int(10) unsigned DEFAULT NULL,
  `description` text DEFAULT NULL,
  `main` tinyint(1) NOT NULL DEFAULT 0,
  `active` tinyint(1) unsigned NOT NULL DEFAULT 1,
  `lastchanged` timestamp NOT NULL DEFAULT current_timestamp() ON UPDATE current_timestamp(),
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=6009 DEFAULT CHARSET=latin1 COLLATE=latin1_swedish_ci;

    PostgreSQL Target Table: chapter

    CREATE TABLE IF NOT EXISTS chapter (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  shortcode VARCHAR(16) UNIQUE,
  title VARCHAR(45),
  description TEXT,
  poc VARCHAR(45),
  email TEXT,
  work VARCHAR(25),
  fax VARCHAR(25),
  website TEXT,
  year_type year_type_enum DEFAULT 'Initiation',
  comments TEXT,
  is_enabled BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  last_changed TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

    Key Changes and Improvements

    1. Primary Key Modernization

    MySQL: Auto-increment integer 

    id int(10) unsigned NOT NULL AUTO_INCREMENT

    PostgreSQL: UUID for better distributed system support 

    id UUID PRIMARY KEY DEFAULT uuid_generate_v4()

    Benefits: - Globally unique identifiers - Better for distributed systems - Enhanced security (non-sequential) - Easier data merging and synchronization

    2. Enhanced Data Chapter

    New Chapteral Fields

    PostgreSQL adds several new fields for better chapter management: 

    shortcode VARCHAR(16) UNIQUE,      -- Short identifier code
poc VARCHAR(45),                   -- Point of contact
year_type year_type_enum DEFAULT 'Initiation',  -- Year type classification

    Status Management Enhancement

    MySQL: Basic active flag 

    active tinyint(1) unsigned NOT NULL DEFAULT 1

    PostgreSQL: Enhanced status tracking 

    is_enabled BOOLEAN DEFAULT true

    3. Contact Information Enhancement

    Phone and Communication

    PostgreSQL adds dedicated contact fields: 

    work VARCHAR(25),                  -- Work phone number
fax VARCHAR(25),                   -- Fax number
email TEXT,                        -- Email address
website TEXT                       -- Website URL

    4. Audit and Timestamp Tracking

    Enhanced Timestamp Management

    MySQL: Basic timestamp tracking 

    lastchanged timestamp NOT NULL DEFAULT current_timestamp() ON UPDATE current_timestamp()

    PostgreSQL: Comprehensive audit trail with timezone support 

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
last_changed TIMESTAMPTZ NOT NULL DEFAULT NOW()

    5. Improved Data Structure

    Description and Comments

    MySQL: Basic description 

    description text DEFAULT NULL

    PostgreSQL: Enhanced description and comments structure 

    description TEXT,
comments TEXT

    Field Mapping Reference

    Complete MySQL to PostgreSQL Field Mapping

    MySQL Field PostgreSQL Field Type Change Notes
    id id INT UUID Primary key conversion
    title title VARCHAR(45) VARCHAR(45) Same size, but enhanced validation
    description description TEXT TEXT Direct mapping
    active is_enabled TINYINT(1) BOOLEAN Status field renamed and type improved
    lastchanged last_changed TIMESTAMP TIMESTAMPTZ Timezone support added
    N/A created_at New TIMESTAMPTZ Creation timestamp tracking

    New Fields (PostgreSQL Only)

    Field Type Purpose
    shortcode VARCHAR(16) UNIQUE Short identifier code for chapters
    poc VARCHAR(45) Point of contact person
    email TEXT Chapter email address
    work VARCHAR(25) Work phone number
    fax VARCHAR(25) Fax number
    website TEXT Chapter website URL
    year_type year_type_enum Year type classification (defaults to 'Initiation')
    comments TEXT Additional comments and notes

    Filtered Fields (Not Migrated)

    MySQL Field Reason Not Migrated
    main_group Used only for linking, not needed in target
    main Legacy field, functionality replaced by other mechanisms

    Migration Logic

    1. Filtering and Selection

    Only chapters (not chapter groups) are migrated:

    SELECT * FROM Donor_Group
WHERE main = 1

    2. UUID Generation and Mapping

    Each MySQL id is mapped to a new PostgreSQL UUID:

    -- Create mapping for reference by other tables
INSERT INTO id_mapping (mysql_table, mysql_id, postgres_id)
SELECT 'donor_group', id, gen_random_uuid()
FROM mysql.Donor_Group WHERE main = 1;

    3. Default Value Application

    -- Set default values during migration
UPDATE chapter
SET
    is_enabled = COALESCE(is_enabled, true),
    year_type = COALESCE(year_type, 'Initiation'),
    title = CASE
        WHEN title IS NULL OR title = ''
        THEN 'Chapter ' || id::TEXT
        ELSE title
    END;

    Chapter Management

    -- Find active chapters
SELECT id, title, shortcode, poc
FROM chapter
WHERE is_enabled = true
ORDER BY title;

-- Search chapters by shortcode
SELECT id, title, description, poc, email
FROM chapter
WHERE shortcode = 'ALPHA'
  AND is_enabled = true;

-- Find chapters by name pattern
SELECT id, title, shortcode, website
FROM chapter
WHERE title ILIKE '%alpha%'
   AND is_enabled = true
ORDER BY title;

-- Chapters by year type
SELECT id, title, year_type, poc
FROM chapter
WHERE year_type = 'Initiation'
  AND is_enabled = true
ORDER BY title;

    Contact Information Queries

    -- Chapters with complete contact info
SELECT id, title, poc, email, work, website
FROM chapter
WHERE is_enabled = true
  AND poc IS NOT NULL
  AND email IS NOT NULL
ORDER BY title;

-- Chapters needing contact update
SELECT id, title, shortcode
FROM chapter
WHERE is_enabled = true
  AND (poc IS NULL OR email IS NULL)
ORDER BY title;

-- Chapters with websites
SELECT id, title, website, email
FROM chapter
WHERE is_enabled = true
  AND website IS NOT NULL
ORDER BY title;

    Administrative Queries

    -- Recently created chapters
SELECT id, title, shortcode, created_at
FROM chapter
WHERE created_at >= CURRENT_DATE - INTERVAL '30 days'
ORDER BY created_at DESC;

-- Recently updated chapters
SELECT id, title, last_changed
FROM chapter
WHERE last_changed >= CURRENT_DATE - INTERVAL '7 days'
ORDER BY last_changed DESC;

    Troubleshooting

    Common Migration Issues

    1. Duplicate Titles

    -- Check for potential title conflicts
SELECT title, COUNT(*) as count
FROM chapter
GROUP BY title
HAVING COUNT(*) > 1;

    2. Duplicate Shortcodes

    -- Check for shortcode conflicts (should be unique)
SELECT shortcode, COUNT(*) as count
FROM chapter
WHERE shortcode IS NOT NULL
GROUP BY shortcode
HAVING COUNT(*) > 1;

    3. Invalid Contact Information

    -- Find chapters with invalid data
SELECT id, title, email, website, work
FROM chapter
WHERE (email IS NOT NULL AND email !~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$')
   OR (website IS NOT NULL AND website !~ '^https?://')
   OR (work IS NOT NULL AND LENGTH(work) < 10);

    4. Missing Required Data

    -- Chapters missing critical information
SELECT id, title, shortcode,
       CASE
         WHEN title IS NULL OR title = '' THEN 'Missing title'
         WHEN shortcode IS NULL OR shortcode = '' THEN 'Missing shortcode'
       END as issue
FROM chapter
WHERE title IS NULL OR title = '' OR shortcode IS NULL OR shortcode = '';
