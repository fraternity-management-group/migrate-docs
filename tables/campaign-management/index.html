    Campaign Management System

    The GMS database includes a comprehensive campaign management system designed to support modern fundraising operations. This system centers around campaigns as the primary organizing unit for prospect management and fundraising activities.

    System Architecture

    Core Design Philosophy

    The campaign management system is built on several key principles:

    1. Campaign-Centric Organization: All prospect activities are organized within specific campaign contexts
    2. Integrated Data Model: Related data is structured in focused, related tables rather than complex normalization
    3. Comprehensive Lifecycle Tracking: Full prospect journey from identification to final outcome
    4. Multi-Dimensional Analysis: Multiple scoring and assessment dimensions for nuanced evaluation
    5. Operational Excellence: Built-in assignment management, action planning, and stewardship tracking

    Campaign Performance Tracking

    -- Get campaign performance overview
SELECT
    c.name,
    c.target_amount,
    COUNT(p.id) as total_prospects,
    SUM(p.suggested_ask_amount) as total_projected,
    (SUM(p.suggested_ask_amount) / c.target_amount) * 100 as projected_percentage,
    AVG(ps.overall_score) as avg_prospect_score
FROM campaign c
LEFT JOIN prospect p ON c.id = p.campaign_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE c.id = :campaign_id
GROUP BY c.id, c.name, c.target_amount;

    Wealth Analysis Queries

    -- Identify high-capacity prospects with underutilized potential
SELECT
    per.display_name,
    wi.net_worth,
    wi.giving_capacity,
    p.suggested_ask_amount,
    CASE
        WHEN wi.giving_capacity = 'OVER_1M' AND p.suggested_ask_amount < 500000 THEN 'UNDERASKED'
        WHEN wi.giving_capacity = '500K_TO_1M' AND p.suggested_ask_amount < 250000 THEN 'UNDERASKED'
        ELSE 'APPROPRIATE'
    END as ask_assessment
FROM prospect p
JOIN person per ON p.person_id = per.id
JOIN prospect_wealth_indicators wi ON p.id = wi.prospect_id
WHERE wi.net_worth IN ('5M_TO_10M', '10M_TO_50M', 'OVER_50M')
ORDER BY wi.net_worth DESC, p.suggested_ask_amount DESC;

    Contact Management Workflow

    Solicitor Assignment and Workload Analysis

    -- Analyze solicitor workload
SELECT
    pci.assigned_solicitor,
    COUNT(*) as total_prospects,
    SUM(p.suggested_ask_amount) as total_ask_amount,
    COUNT(CASE WHEN p.priority_level = 'HIGH' THEN 1 END) as high_priority_count,
    COUNT(CASE WHEN pci.contact_status = 'FOLLOW_UP_NEEDED' THEN 1 END) as follow_ups_needed,
    AVG(ps.overall_score) as avg_prospect_score
FROM prospect p
JOIN prospect_campaign_interaction pci ON p.id = pci.prospect_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE pci.assigned_solicitor IS NOT NULL
GROUP BY pci.assigned_solicitor
ORDER BY total_ask_amount DESC;

    Assessment Reporting

    -- Generate assessment summary report
SELECT
    per.display_name,
    p.prospect_status,
    p.priority_level,
    pac.loyalty_assessment_comment,
    pac.leadership_assessment_comment,
    pac.wealth_assessment_comment,
    ps.overall_score,
    pci.assigned_solicitor,
    pci.next_action_date
FROM prospect p
JOIN person per ON p.person_id = per.id
LEFT JOIN prospect_assessment_comments pac ON p.id = pac.prospect_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
LEFT JOIN prospect_campaign_interaction pci ON p.id = pci.prospect_id
WHERE p.campaign_id = :campaign_id
  AND p.prospect_status IN ('CULTIVATING', 'SOLICITATION_READY')
ORDER BY ps.overall_score DESC;

    Reporting and Analytics

    Campaign Dashboard Metrics

    -- Executive campaign dashboard
SELECT
    c.name as campaign_name,
    c.target_amount,
    COUNT(p.id) as total_prospects,
    COUNT(CASE WHEN p.prospect_status = 'COMMITTED' THEN 1 END) as committed_count,
    COUNT(CASE WHEN p.prospect_status = 'PENDING_DECISION' THEN 1 END) as pending_count,
    SUM(CASE WHEN p.prospect_status = 'COMMITTED' THEN p.suggested_ask_amount ELSE 0 END) as committed_amount,
    SUM(p.suggested_ask_amount) as total_projected,
    (COUNT(CASE WHEN p.prospect_status = 'COMMITTED' THEN 1 END)::DECIMAL /
     NULLIF(COUNT(p.id), 0)) * 100 as commitment_rate,
    AVG(ps.overall_score) as avg_prospect_quality
FROM campaign c
LEFT JOIN prospect p ON c.id = p.campaign_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE c.campaign_type = 'CAPITAL'
GROUP BY c.id, c.name, c.target_amount
ORDER BY commitment_rate DESC;

    Prospect Pipeline Analysis

    -- Analyze prospect movement through pipeline
SELECT
    p.prospect_status,
    COUNT(*) as prospect_count,
    SUM(p.suggested_ask_amount) as total_potential,
    AVG(ps.overall_score) as avg_score,
    AVG(EXTRACT(days FROM (CURRENT_DATE - p.created_at))) as avg_days_in_pipeline
FROM prospect p
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE p.campaign_id = :campaign_id
GROUP BY p.prospect_status
ORDER BY
    CASE p.prospect_status
        WHEN 'IDENTIFIED' THEN 1
        WHEN 'QUALIFIED' THEN 2
        WHEN 'CULTIVATING' THEN 3
        WHEN 'SOLICITATION_READY' THEN 4
        WHEN 'PENDING_DECISION' THEN 5
        WHEN 'COMMITTED' THEN 6
        ELSE 7
    END;

    Best Practices

    Data Quality Guidelines

    1. Consistent Enum Usage: Always use standardized enum values for status and classification fields
    2. Regular Scoring Updates: Update prospect scores monthly or when significant new information is available
    3. Action Date Management: Keep next action dates current and realistic
    4. Comment Documentation: Provide detailed, professional assessment comments for major decisions

    Performance Optimization

    1. Index Usage: Leverage the comprehensive indexing strategy for optimal query performance
    2. Campaign Filtering: Always filter by campaign_id in prospect queries when possible
    3. Batch Operations: Use batch operations for bulk updates to prospect data
    4. View Utilization: Use performance views for complex reporting and analysis

    Security Considerations

    1. Row-Level Security: Implement appropriate row-level security policies based on chapter membership
    2. Comment Auditing: Maintain audit trails for assessment comments and scoring changes
    3. Access Controls: Restrict sensitive wealth information to authorized users only
    4. Data Retention: Implement appropriate data retention policies for historical campaign data

    This campaign management system provides a comprehensive foundation for modern fundraising operations, supporting the full prospect lifecycle from identification through stewardship, with robust reporting and analytics capabilities.
