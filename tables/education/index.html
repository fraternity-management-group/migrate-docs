    Education Table Migration

    Overview

    The Education table migration transforms the MySQL Schools table into a comprehensive PostgreSQL education tracking system with enhanced degree classification, institution management, and academic achievement tracking.

    Note: This documentation reflects the current state of the education table. The is_graduated column has been removed as graduation status can be inferred from the presence of a grad_date value.

    Schema Transformation

    MySQL Source Table

    • Schools - Basic education records

    PostgreSQL Target Table

    • education - Enhanced education tracking with standardized degrees and institutions

    Field Mappings

    Core Education Fields

    MySQL Field PostgreSQL Field Type Change Notes
    sid id INT → UUID Primary key conversion
    donor_id person_id INT → UUID Foreign key to person table
    college institution_name VARCHAR(45) → TEXT Institution name field
    degree degree VARCHAR(45) → TEXT Original degree text preserved
    N/A degree_type N/A → degree_type_enum Derived from degree field
    grad_date grad_date YEAR(4) → INTEGER Graduation year
    student_id student_id VARCHAR(45) → TEXT Student ID preservation

    Enhanced Fields (New in PostgreSQL)

    Field Type Purpose
    institution_type institution_type_enum Type of educational institution (derived)
    degree_level degree_level_enum Academic level classification (derived)
    major VARCHAR(100) Academic major field
    minor VARCHAR(100) Academic minor field
    concentration VARCHAR(100) Area of concentration/specialization
    honors honors_enum Academic honors received
    gpa DECIMAL(5,2) Grade point average
    gpa_scale DECIMAL(3,1) GPA scale (default 4.0)
    start_date DATE Estimated program start date
    end_date DATE Program completion date
    thesis_title TEXT Thesis or dissertation title
    advisor VARCHAR(100) Academic advisor name
    last_changed TIMESTAMP Last modification timestamp

    Table Schema

    Current PostgreSQL Education Table

    CREATE TABLE IF NOT EXISTS education (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  person_id UUID NOT NULL REFERENCES person(id) ON DELETE CASCADE,
  degree TEXT,
  degree_type degree_type_enum,
  institution_name TEXT,
  institution_type institution_type_enum,
  degree_level degree_level_enum,
  major VARCHAR(100),
  minor VARCHAR(100),
  concentration VARCHAR(100),
  honors honors_enum,
  gpa DECIMAL(5,2),
  gpa_scale DECIMAL(3,1) DEFAULT 4.0,
  grad_date INTEGER,
  start_date DATE,
  end_date DATE,
  student_id TEXT,
  thesis_title TEXT,
  advisor VARCHAR(100),
  last_changed TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT valid_date_order CHECK (start_date <= COALESCE(end_date, CURRENT_DATE)),
  CONSTRAINT valid_grad_date CHECK (grad_date >= start_date),
  CONSTRAINT valid_gpa CHECK (gpa >= 0 AND gpa <= gpa_scale)
);

    Constraints

    1. Date Constraints
    2. valid_date_order: Start date must be before or equal to end date

    3. valid_grad_date: Graduation year must be after start date

    4. GPA Constraints

    5. valid_gpa: GPA must be between 0 and the specified GPA scale
    6. GPA scale defaults to 4.0 but can be adjusted for different systems

    Indexes and Performance

    Primary Indexes

    -- Primary key
CREATE UNIQUE INDEX idx_education_pk ON education(id);

-- Foreign key indexes
CREATE INDEX idx_education_person_id ON education(person_id);

    Performance Indexes

    -- Institution-based queries
CREATE INDEX idx_education_institution ON education(institution_name);
CREATE INDEX idx_education_institution_type ON education(institution_type);

-- Degree-based queries
CREATE INDEX idx_education_degree_type ON education(degree_type);
CREATE INDEX idx_education_degree_level ON education(degree_level);

-- Year-based queries
CREATE INDEX idx_education_grad_date ON education(grad_date);

-- Composite indexes for common queries
CREATE INDEX idx_education_person_level ON education(person_id, degree_level);
CREATE INDEX idx_education_institution_year ON education(institution_name, grad_date);

    Common Queries

    Alumni by Institution

    SELECT
    institution_name,
    COUNT(*) as alumni_count,
    COUNT(DISTINCT person_id) as unique_alumni
FROM education
GROUP BY institution_name
ORDER BY alumni_count DESC;

    Degree Distribution

    SELECT
    degree_type,
    degree_level,
    COUNT(*) as count,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) as percentage
FROM education
GROUP BY degree_type, degree_level
ORDER BY count DESC;

    Recent Graduates

    SELECT
    p.first_name,
    p.last_name,
    e.institution_name,
    e.degree_type,
    e.major,
    e.grad_date
FROM education e
JOIN person p ON e.person_id = p.id
WHERE e.grad_date >= EXTRACT(YEAR FROM CURRENT_DATE) - 5
ORDER BY e.grad_date DESC, e.institution_name;

    Academic Honors Recipients

    SELECT
    p.first_name,
    p.last_name,
    e.institution_name,
    e.degree_type,
    e.honors,
    e.gpa
FROM education e
JOIN person p ON e.person_id = p.id
WHERE e.honors IS NOT NULL
ORDER BY e.grad_date DESC;

    Multi-Degree Holders

    SELECT
    p.first_name,
    p.last_name,
    COUNT(*) as degree_count,
    STRING_AGG(e.degree_type || ' - ' || e.institution_name, '; ' ORDER BY e.grad_date) as degrees
FROM education e
JOIN person p ON e.person_id = p.id
GROUP BY p.id, p.first_name, p.last_name
HAVING COUNT(*) > 1
ORDER BY degree_count DESC;

    Validation Rules

    1. Degree Validation
    2. Degree type must match approved enum values
    3. Degree level must correspond to degree type

    4. Professional degrees require appropriate institutions

    5. Date Validation

    6. Graduation year cannot be in the future
    7. Start date must precede end date (if both provided)

    8. Graduation year must be after start date (if both provided)

    9. GPA Validation

    10. GPA must be between 0.0 and the specified GPA scale
    11. GPA precision limited to 2 decimal places
    12. GPA scale defaults to 4.0 but can be adjusted

    13. Null values allowed for unknown GPAs

    14. Institution Validation

    15. Institution names can be free text
    16. Institution type must match approved enum values
    17. International institutions properly classified

    Data Cleanup During Migration

    1. Institution Name Standardization 

      -- Standardize common institution name variations
UPDATE education_temp SET institution_name = 'University of California, Berkeley'
WHERE institution_name IN ('UC Berkeley', 'Cal', 'Berkeley');

UPDATE education_temp SET institution_name = 'Massachusetts Institute of Technology'
WHERE institution_name IN ('MIT', 'M.I.T.');

    2. Degree Abbreviation Expansion 

      -- Expand common degree abbreviations
UPDATE education_temp SET degree_type = 'bachelor'
WHERE degree IN ('BS', 'BA', 'BSc', 'Bachelor');

UPDATE education_temp SET degree_type = 'master'
WHERE degree IN ('MS', 'MA', 'MSc', 'Master');

    3. Data Anomaly Detection 

      -- Flag unusual graduation years for review
SELECT * FROM education_temp
WHERE grad_date < 1950 OR grad_date > EXTRACT(YEAR FROM CURRENT_DATE) + 10;

-- Flag unusual GPA values
SELECT * FROM education_temp
WHERE gpa > gpa_scale OR gpa < 0.0;

    Institutional Partnerships

    Alumni Engagement Enhancement

    1. Institutional Grouping 

      -- Create institution groups for targeted communications
CREATE VIEW institution_groups AS
SELECT
    CASE
        WHEN institution_type = 'public_university' THEN 'Public Universities'
        WHEN institution_type = 'private_university' THEN 'Private Universities'
        WHEN institution_type = 'community_college' THEN 'Community Colleges'
        ELSE 'Other Institutions'
    END as institution_group,
    institution_name,
    COUNT(*) as alumni_count
FROM education
GROUP BY institution_group, institution_name;

    2. Class Year Reunions 

      -- Identify reunion year classes
SELECT
    grad_date,
    institution_name,
    COUNT(*) as class_size,
    CASE
        WHEN (EXTRACT(YEAR FROM CURRENT_DATE) - grad_date) % 5 = 0
        THEN 'Reunion Year'
        ELSE 'Regular Year'
    END as reunion_status
FROM education
WHERE grad_date IS NOT NULL
GROUP BY grad_date, institution_name
HAVING COUNT(*) >= 3;

    Troubleshooting

    Common Migration Issues

    1. Invalid Degree Types 

      Error: invalid input value for enum degree_type_enum
Solution: Update degree type mapping to handle all variations

    2. GPA Scale Variations 

      Error: GPA values outside expected range
Solution: Implement GPA scale conversion for different systems

    3. Institution Name Duplicates 

      Error: Multiple variations of same institution name
Solution: Implement institution name standardization lookup

    4. Missing Person References 

      Error: foreign key constraint violation
Solution: Ensure all donor_id values exist in person table
