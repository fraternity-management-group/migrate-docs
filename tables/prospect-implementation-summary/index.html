    Campaign Management and Prospect Tracking System

    This document describes the integrated campaign management and prospect tracking system implemented in migration 000008_prospect.up.sql, which replaced the previous normalized approach with a comprehensive campaign-centric design.

    System Overview

    The new system centers around campaigns as the primary organizing unit for prospect management, with prospects being tracked within specific campaign contexts. This approach better reflects real-world fundraising operations where prospects are managed within focused campaign efforts.

    Core Design Principles

    1. Campaign-Centric: All prospects are associated with specific campaigns
    2. Integrated Data Model: Related prospect data is organized in focused, related tables rather than complex normalization
    3. Comprehensive Tracking: Full lifecycle tracking from initial assessment to final outcomes
    4. Flexible Scoring: Multiple scoring dimensions for nuanced prospect evaluation

    Schema Architecture

    1. Campaign Management (campaign table)

    1. Campaign Management (campaign table)

    Central organizing unit for all prospect activities:

    CREATE TABLE campaign (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    campaign_type campaign_type_enum NOT NULL,
    start_date DATE,
    end_date DATE,
    target_amount DECIMAL(15,2),
    chapter_id UUID REFERENCES chapter(id) ON DELETE CASCADE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

    Key Features: - Campaign lifecycle management with start/end dates - Financial targeting with target amounts - Campaign type categorization (CAPITAL, ANNUAL, SPECIAL, FEASIBILITY) - Chapter-specific campaign organization

    2. Prospect Core Data (prospect table)

    Main prospect information with campaign association:

    CREATE TABLE prospect (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    person_id UUID NOT NULL REFERENCES person(id) ON DELETE CASCADE,
    campaign_id UUID NOT NULL REFERENCES campaign(id) ON DELETE CASCADE,
    prospect_status prospect_status_enum DEFAULT 'IDENTIFIED',
    priority_level priority_level_enum DEFAULT 'MEDIUM',
    -- Financial fields
    estimated_capacity_range capacity_range_enum,
    suggested_ask_amount DECIMAL(15,2),
    gift_range_minimum DECIMAL(15,2),
    gift_range_maximum DECIMAL(15,2),
    -- Assessment fields
    likelihood_to_give likelihood_enum,
    cultivation_level cultivation_level_enum DEFAULT 'INITIAL',
    -- Notes and comments
    prospect_notes TEXT,
    internal_notes TEXT
);

    Key Features: - Direct campaign association for all prospects - Comprehensive status tracking throughout prospect lifecycle - Financial capacity assessment with ranges and specific ask amounts - Cultivation progress tracking

    3. Wealth Assessment (prospect_wealth_indicators table)

    Detailed wealth and capacity indicators:

    CREATE TABLE prospect_wealth_indicators (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    prospect_id UUID NOT NULL REFERENCES prospect(id) ON DELETE CASCADE,
    annual_income income_range_enum,
    net_worth networth_range_enum,
    giving_capacity giving_capacity_enum,
    real_estate_value DECIMAL(15,2),
    business_interests TEXT,
    investment_assets TEXT,
    other_assets TEXT,
    source_of_wealth TEXT,
    wealth_verification_notes TEXT
);

    Key Features: - Standardized income and net worth enumeration - Detailed asset tracking capabilities - Wealth source documentation - Verification notes for due diligence

    4. Prospect Scoring (prospect_scoring table)

    Multi-dimensional scoring system:

    CREATE TABLE prospect_scoring (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    prospect_id UUID NOT NULL REFERENCES prospect(id) ON DELETE CASCADE,
    wealth_rating_score DECIMAL(3,2),           -- 0.00 to 5.00
    inclination_score DECIMAL(5,2),             -- 0.00 to 100.00
    overall_score DECIMAL(5,2),                 -- 0.00 to 100.00
    external_rating VARCHAR(50),
    research_score DECIMAL(5,2),                -- 0.00 to 100.00
    last_score_update TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

    Key Features: - Multiple scoring dimensions (wealth, inclination, research, overall) - External rating integration capability - Timestamp tracking for score updates - Standardized 0-100 percentage scoring

    5. Campaign Interaction Tracking (prospect_campaign_interaction table)

    Detailed campaign activity tracking:

    CREATE TABLE prospect_campaign_interaction (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    prospect_id UUID NOT NULL REFERENCES prospect(id) ON DELETE CASCADE,
    contact_status contact_status_enum DEFAULT 'NOT_CONTACTED',
    assigned_solicitor VARCHAR(255),
    assigned_section VARCHAR(255),
    information_package_sent_date DATE,
    last_contact_date DATE,
    next_action_date DATE,
    next_action_type VARCHAR(100),
    contact_frequency contact_frequency_enum,
    stewardship_level stewardship_level_enum
);

    Key Features: - Complete contact lifecycle tracking - Solicitor and section assignment management - Action planning with next steps - Stewardship level management - Contact frequency preferences

    6. Assessment Comments (prospect_assessment_comments table)

    Detailed assessment documentation:

    CREATE TABLE prospect_assessment_comments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    prospect_id UUID NOT NULL REFERENCES prospect(id) ON DELETE CASCADE,
    loyalty_assessment_comment TEXT,
    leadership_assessment_comment TEXT,
    wealth_assessment_comment TEXT,
    cultivation_assessment_comment TEXT,
    general_assessment_comment TEXT,
    low_priority_reason TEXT,
    comment_date TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    created_by VARCHAR(255)
);

    Key Features: - Structured comment categorization - Assessment-specific commentary - Audit trail with timestamps and user tracking - Low priority reasoning documentation

    Enhanced Enum System

    The new system uses comprehensive enumerations for consistent data quality:

    Campaign Types

    CREATE TYPE campaign_type_enum AS ENUM (
    'CAPITAL', 'ANNUAL', 'SPECIAL', 'FEASIBILITY', 'ENDOWMENT'
);

    Prospect Status Lifecycle

    CREATE TYPE prospect_status_enum AS ENUM (
    'IDENTIFIED', 'QUALIFIED', 'CULTIVATING', 'SOLICITATION_READY',
    'PENDING_DECISION', 'COMMITTED', 'DECLINED', 'DEFERRED', 'INACTIVE'
);

    Financial Capacity Ranges

    CREATE TYPE capacity_range_enum AS ENUM (
    'UNDER_1K', '1K_TO_5K', '5K_TO_10K', '10K_TO_25K', '25K_TO_50K',
    '50K_TO_100K', '100K_TO_250K', '250K_TO_500K', '500K_TO_1M', 'OVER_1M'
);

CREATE TYPE networth_range_enum AS ENUM (
    'UNDER_100K', '100K_TO_500K', '500K_TO_1M', '1M_TO_5M',
    '5M_TO_10M', '10M_TO_50M', 'OVER_50M'
);

CREATE TYPE income_range_enum AS ENUM (
    'UNDER_50K', '50K_TO_100K', '100K_TO_250K', '250K_TO_500K',
    '500K_TO_1M', 'OVER_1M'
);

    Contact and Cultivation Management

    CREATE TYPE contact_status_enum AS ENUM (
    'NOT_CONTACTED', 'INITIAL_CONTACT', 'FOLLOW_UP_NEEDED',
    'IN_CULTIVATION', 'READY_FOR_ASK', 'ASKED', 'COMMITTED',
    'DECLINED', 'ON_HOLD'
);

CREATE TYPE cultivation_level_enum AS ENUM (
    'INITIAL', 'BASIC', 'DEVELOPING', 'ADVANCED', 'READY', 'ONGOING'
);

CREATE TYPE stewardship_level_enum AS ENUM (
    'NONE', 'BASIC', 'ENHANCED', 'VIP', 'BOARD_LEVEL'
);

    Migration Process Flow:

    1. Campaign Creation: Ensures default campaign exists for prospect migration
    2. Core Prospect Data: Migrates main prospect information to prospect table
    3. Wealth Indicators: Transfers wealth assessment data to prospect_wealth_indicators
    4. Scoring Data: Converts scoring information for prospect_scoring table
    5. Campaign Interactions: Maps contact and solicitor data to prospect_campaign_interaction
    6. Assessment Comments: Preserves comment data in prospect_assessment_comments

    Usage Examples

    Campaign Management

    -- Create a new campaign
INSERT INTO campaign (name, description, campaign_type, target_amount, chapter_id)
VALUES ('Capital Campaign 2025', 'Major building expansion', 'CAPITAL', 5000000.00, :chapter_id);

-- Get campaign progress
SELECT c.name, COUNT(p.id) as prospect_count,
       SUM(p.suggested_ask_amount) as total_potential
FROM campaign c
LEFT JOIN prospect p ON c.id = p.campaign_id
GROUP BY c.id, c.name;

    Prospect Analysis

    -- Find high-priority prospects ready for cultivation
SELECT p.person_id, per.display_name, p.suggested_ask_amount,
       wi.net_worth, ps.overall_score
FROM prospect p
JOIN person per ON p.person_id = per.id
LEFT JOIN prospect_wealth_indicators wi ON p.id = wi.prospect_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE p.priority_level = 'HIGH'
  AND p.cultivation_level IN ('DEVELOPING', 'ADVANCED')
ORDER BY ps.overall_score DESC, p.suggested_ask_amount DESC;

    Solicitor Workload

    -- Analyze solicitor assignments and workload
SELECT pci.assigned_solicitor,
       COUNT(*) as total_prospects,
       SUM(p.suggested_ask_amount) as total_ask_amount,
       AVG(ps.overall_score) as avg_prospect_score
FROM prospect p
JOIN prospect_campaign_interaction pci ON p.id = pci.prospect_id
LEFT JOIN prospect_scoring ps ON p.id = ps.prospect_id
WHERE pci.assigned_solicitor IS NOT NULL
GROUP BY pci.assigned_solicitor
ORDER BY total_ask_amount DESC;
